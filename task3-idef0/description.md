# Описание функциональной модели гостиничного предприятия

Для функционального моделирования деятельности гостиничного предприятия была использована нотация IDEF0.

На первом этапе построена контекстная диаграмма A-0, отражающая деятельность гостиничного предприятия как единую систему.

Входами модели являются:

- заявки на бронирование;
- данные гостей;
- дополнительные запросы клиентов.

Управляющими воздействиями выступают:

- стандарты обслуживания;
- тарифная политика гостиницы.

Механизмами выполнения функций являются:

- персонал гостиницы;
- номерной фонд.

Результатами работы системы являются:

- услуги размещения;
- дополнительные услуги;
- отчетная документация.

После построения контекстной диаграммы была выполнена декомпозиция основной функции.

В результате были выделены три взаимосвязанные функции:

1. Бронирование номера.
2. Регистрация и размещение гостей.
3. Окончательный расчет за проживание и услуги.

Функция бронирования обеспечивает прием заявок и подтверждение возможности размещения гостя.

Функция регистрации и размещения включает проверку данных, оформление документов и предоставление номера для проживания.

Функция окончательного расчета предназначена для учета оказанных услуг, проведения расчетов с клиентом и формирования отчетной документации.

Построенная модель позволяет представить структуру деятельности гостиничного предприятия и определить взаимосвязи между основными функциями системы.
