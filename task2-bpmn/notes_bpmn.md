# Конспект по нотации BPMN 2.0

## Назначение BPMN

BPMN (Business Process Model and Notation) представляет собой нотацию графического моделирования бизнес-процессов. Основная цель BPMN заключается в создании понятных и унифицированных схем процессов, которые могут использоваться как специалистами предметной области, так и разработчиками информационных систем.

## Основные элементы BPMN

### События (Events)

События используются для обозначения начала, окончания или промежуточного состояния процесса.

Основные виды событий:

- стартовое событие;
- промежуточное событие;
- конечное событие.

### Задачи (Tasks)

Задача представляет собой отдельное действие, выполняемое участником процесса.

Примеры задач:

- обработка заявки;
- проверка данных;
- оформление документов;
- подтверждение бронирования.

### Шлюзы (Gateways)

Шлюзы используются для организации ветвления и объединения потоков процесса.

Основные виды шлюзов:

- исключающий шлюз (XOR);
- параллельный шлюз (AND);
- включающий шлюз (OR).

### Потоки управления (Sequence Flow)

Потоки управления показывают последовательность выполнения действий внутри процесса.

### Дорожки ответственности (Lanes)

Дорожки используются для разделения ответственности между участниками процесса.

В модели гостиничного предприятия используются следующие дорожки:

- Гость;
- Служба бронирования;
- Служба регистрации и размещения.

## Преимущества BPMN

Использование BPMN позволяет:

- визуализировать бизнес-процессы;
- определить участников процесса;
- установить последовательность операций;
- выявить возможные проблемы процесса;
- подготовить основу для автоматизации деятельности организации.
